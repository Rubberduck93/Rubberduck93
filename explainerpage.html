Parking in Copenhagen

Motivation

The visualization consists of 4 different datasets:

  • parking.geojson - All available parking spaces in Copenhagen.
  • kbh.geojson - Map of Copenhagen and all the towns within.
  • zoner.geojson - All Parkingzones in Copenhagen.
  • phuse.geojson - All Parkinghouses in Copenhagen.

    • I chose these datasets because they all share valuable information regarding parking in Copenhagen.

    The overall goal was to give the end user an overview of all the parking possibilities in one visualization.

    Basic stats

    All of the datasets were found data.kk.dk/dataset and they are all geojson datasets. I have not used any kind of cleaning nor preprocessing for the datasets.
    Most of them were clean enough and small enough to handle. It may have been a mistake not to preprocess the parking.geojson because of its size (14.33 MB).
    The most extracted information from the geojsons were the coordinates.
    From the parking.geojson the coordinates were drawn using D3's Path. The brush function counting the parkingspaces were located here.
    From the kbh.geojson the coordinates were too drawn using D3's Path, else the different towns name and population numbers were used for text and coloring.

    Visualizations

    There is a lot of feedback in the visualization, which is important for the end user.
    The mouseover effect is especially important, because it allows most information to be hidden.
    Only showing information when the user which to get information creates a more relaxed enviroment by not overflowing the visualization and end user with unneccesary information.

    Discussion

    Good:
    I feel the visualization and the explainerpage turned out nicely as a whole. There are links in between each other and links to the Repository. The visualization is simple but compact as it contains more information than meets the eye.
    Bad:
    The parking.geojson file is huge (14.33 MB). This causes the webpage to slow down whenever it has to read from the parking.geojson file. I should have used some preprocessing here.
