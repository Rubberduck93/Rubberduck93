Parking in Copenhagen

Motivation

The visualization consists of 4 different datasets:

  • parking.geojson - All available parking spaces in Copenhagen.
  • kbh.geojson - Map of Copenhagen and all the towns within.
  • zoner.geojson - All Parkingzones in Copenhagen.
  • phuse.geojson - All Parkinghouses in Copenhagen.

    • I chose these datasets because they all share valuable information regarding parking in Copenhagen.

    The overall goal was to give the end user an overview of all the parking possibilities in one visualization.

    Basic stats

    All of the datasets are geojson datasets found here: data.kk.dk/dataset. I have not used any kind of cleaning nor preprocessing for the datasets.
    Most of them were clean enough and small enough to handle. It may have been a mistake not to preprocess the parking.geojson because of its size (14.33 MB).
    The most extracted information from the geojsons were the coordinates.
    From the parking.geojson the coordinates were drawn using D3's Path. The brush function counting the parkingspaces were located here.
    From the kbh.geojson the coordinates were too drawn using D3's Path, else the different towns name and population numbers were used for text and coloring.
    A mouse-over the different towns will highlight and show more details.
    From the zoner.geojson the coordinates were too drawn using D3's Path and a mouse-over will highlight and show closer detains regarding the zones.
    From the phuse.geojson is drawn text with the respective x and y coordinates. The size of the text drawn depends on the size of the individual parkinghouse.
    The order in which each geojson entered the visualization turned out to be very important for accessing data after all datasets were drawn.

    Genre

    Visual Narrative

  • Consistent Visual Platform - The visual platform essentially stays as it is. All of the end users manipulations reverses back to its original form once it is no longer used.
  • Feature Distinction - The towns are colored with different shade of green depending on the population within and the edge is black. The zones are drawn on top with a low opacity.
  • Animated Transitions - The towns, zones and clickable text (links), animates to a different color when mouse-over, same with mouse-out.

    • Narrative Structure

  • parking.geojson - All available parking spaces in Copenhagen.
  • Hover Highlighting / Details - Map of Copenhagen and all the towns within.
  • Captions / Headlines and Annotations - Basic

    • Visualizations

    There is a lot of feedback in the visualization, which is important for the end user.
    The mouseover effect is especially important, because it allows most information to be hidden.
    Only showing information when the user which to get information creates a more relaxed enviroment by not overflowing the visualization and end user with unneccesary information.

    Discussion

    Good:
    I feel the visualization and the explainerpage turned out nicely as a whole. There are links in between eachother and links to the Github Repository. The visualization is simple but compact as it contains more information than meets the eye.
    Bad:
    The parking.geojson file is huge (14.33 MB). This causes the webpage to slow down whenever it has to read from the parking.geojson file. I should have used some preprocessing here.
